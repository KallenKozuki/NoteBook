# LeetCode总结

## 哈希表

+ 哈希表操作的时间复杂度是 ***O(1)***
+ ``hashmap[key]``可以直接访问对应的``value``值，如果该``key``不存在，则会创建该``key``
+ 关于哈希表容器的选择
  + 一般来说是选择``vector``或者``map``
  + ``vector``因为是连续内存空间，访问快，但``key``值只能是``int``类型
  + ``map``空间非连续，由指针链接，故访问慢，存入数据后会自动排序，``key``值可以是任意数据类型

## 递归

+ 实现递归的三个条件
  + 大问题能拆解成两个子问题
  + 子问题的求解方式和大问题一样
  + 存在最小子问题

+ 递归三个核心步骤
  + 分析问题、寻找递归：明确递归函数需要做到的功能
  + 设置边界、控制递归：明确递归的结束条件
  + 设计函数、确定参数：找到相邻的两层递归之间的关系（确定函数的关系）
